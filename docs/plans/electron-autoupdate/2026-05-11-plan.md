# 1. Context

**Problem:**
Cần xây dựng một desktop app demo sử dụng Electron.js + React + TypeScript + shadcn/ui + Tailwind để minh hoạ cơ chế auto-update qua GitHub Releases. App phải vừa hoạt động đúng về tính năng auto-update, vừa thể hiện design system từ DESIGN.md (Figma-style: monochrome core, pastel color-block sections, pill CTAs).

**Affected Modules:**
- `electron/main` — Main process: quản lý window, IPC, tích hợp electron-updater
- `electron/preload` — Bridge IPC an toàn giữa main và renderer
- `src/renderer` — React UI với shadcn/ui, Tailwind, auto-update UI
- `electron-builder config` — Cấu hình build & publish lên GitHub Releases

**Non-Goals:**
- Không triển khai backend/server riêng — hoàn toàn dựa vào GitHub Releases
- Không xây dựng full marketing site — chỉ đủ các UI sections để demo design system
- Không dark mode (DESIGN.md xác nhận marketing site không có dark theme)
- Không authentication/multi-tenant

# 2. Constraints

**Language:** `TypeScript`
**Architecture:** `Electron (Main + Preload + Renderer) / React Component Architecture`

**Rules:**
- Không dùng `nodeIntegration: true` — dùng contextBridge + IPC cho mọi giao tiếp main ↔ renderer
- Tất cả secrets (GitHub token) qua biến môi trường, không hardcode

**Performance Budgets:**

- Update check response: hiển thị trạng thái trong < 500ms sau khi click

# 3. Conventions

**Naming:**
- Components React: `PascalCase` — `UpdateNotification`, `ColorBlockSection`
- Hooks: `use` prefix — `useAutoUpdate`, `useUpdateStatus`
- IPC channels: `kebab-case` — `check-for-updates`, `download-update`, `install-update`
- Constants: `UPPER_SNAKE_CASE` — `UPDATE_CHANNEL`, `APP_VERSION`
- Files: `kebab-case.ts` cho utilities, `PascalCase.tsx` cho components

**Error Handling:**
- IPC handlers wrap logic trong try/catch, gửi error object về renderer qua IPC
- Renderer hiển thị user-friendly message, không expose stack trace
- electron-updater errors bắt tại `autoUpdater.on('error')`, log + notify renderer

**Logging:**
- Main process: `console.log` / `electron-log` với prefix `[main]`, `[updater]`
- Renderer: không log update internals — chỉ nhận state từ IPC

**Testing:**
- Framework: Jest + React Testing Library
- Naming: `should_<expected>_when_<condition>`
- Không unit test electron-updater trực tiếp — test IPC handler mocks

**Task Size:**
- Max 200 LOC per task (excluding tests)

# 4. Contracts

## Interface: `UpdaterIPC`

```
// Renderer → Main (invoke)
window.electronAPI.checkForUpdates(): Promise<UpdateCheckResult>
  - pre: app is running in production (không gọi trong dev sẽ trả mock)
  - post: trả về UpdateCheckResult với status và version info
  - throws: never — lỗi được wrap vào UpdateCheckResult.error

window.electronAPI.downloadUpdate(): Promise<void>
  - pre: checkForUpdates() đã trả về status='available'
  - post: download bắt đầu, progress events được emit qua onDownloadProgress
  - throws: never

window.electronAPI.installAndRestart(): void
  - pre: download đã complete (status='ready')
  - post: app restart và cài bản mới
  - throws: never

// Main → Renderer (on / listener)
window.electronAPI.onUpdateStatus(callback: (status: UpdateStatus) => void): void
  - Renderer đăng ký nhận stream events từ main process
  - Gọi cleanup() để remove listener khi component unmount

window.electronAPI.getAppVersion(): Promise<string>
  - post: returns semver string (e.g. "1.0.0")
```

## Data Structures

```
UpdateCheckResult {
  status:       UpdateStatus      — current update state
  version?:     string            — available version nếu có update
  releaseNotes?: string           — release notes từ GitHub
  error?:        string           — error message nếu check thất bại
}

UpdateStatus = 
  | 'idle'           — chưa check
  | 'checking'       — đang check
  | 'available'      — có bản mới
  | 'not-available'  — đang dùng bản mới nhất
  | 'downloading'    — đang tải
  | 'ready'          — đã tải xong, sẵn sàng install
  | 'error'          — có lỗi

DownloadProgress {
  percent:   number   — 0–100
  bytesPerSecond: number
  transferred: number
  total: number
}
```

# 5. Target Architecture

**Components:**
- `electron/main/index.ts` — Entry point, tạo BrowserWindow, đăng ký IPC handlers
- `electron/main/updater.ts` — Đóng gói electron-updater logic, emit events
- `electron/preload/index.ts` — contextBridge expose `window.electronAPI`
- `src/App.tsx` — Root React component, layout chính
- `src/components/UpdatePanel` — UI panel auto-update (check, progress, install)
- `src/components/layout/*` — TopNav, ColorBlockSection, Footer theo DESIGN.md

**Interaction Flow:**

```
Renderer (React)
  → window.electronAPI.checkForUpdates()  [IPC invoke]
    → main: ipcMain.handle('check-for-updates')
      → updater.checkForUpdates()
        → autoUpdater.checkForUpdates()
  ← UpdateCheckResult { status: 'checking' }

autoUpdater events (main process)
  → updater emits UpdateStatus events via ipcMain
    → webContents.send('update-status', status)
      → preload: onUpdateStatus callback fires
        → React state update → UI re-renders

User clicks "Install & Restart"
  → window.electronAPI.installAndRestart()  [IPC send]
    → main: autoUpdater.quitAndInstall()
```

**Key Decisions:**
- `contextIsolation: true` + `nodeIntegration: false` — bảo mật IPC qua contextBridge
- electron-updater chỉ khởi tạo trong production để tránh lỗi dev mode
- UpdatePanel component là self-contained, dùng custom hook `useAutoUpdate`
- Design tokens implement bằng CSS variables trong `globals.css`, không hardcode màu

## TASK-005: Auto-Update Hook + UI Panel

**Description:**
Implement `useAutoUpdate` custom hook quản lý update state machine, và `UpdatePanel` component hiển thị toàn bộ auto-update flow: idle → checking → available/not-available → downloading (progress) → ready → install.

**Input:**
- TASK-002: `window.electronAPI` (IPC interface)
- TASK-003: design token classes

**Output:** `UpdatePanel` component — consumed by TASK-006

**Files:**
- `src/hooks/useAutoUpdate.ts` — **create**: state machine, IPC calls, cleanup
- `src/components/UpdatePanel/index.tsx` — **create**: UI cho mọi update states
- `src/components/UpdatePanel/ProgressBar.tsx` — **create**: download progress bar

**Responsibilities:**
- Implement `UpdateStatus` state machine với transitions hợp lệ
- Cleanup IPC listener khi component unmount (tránh memory leak)
- `ProgressBar` dùng shadcn `Progress` component, style theo design tokens
- Hiển thị version hiện tại và version mới nếu available

**State Machine:**
```
idle → [checkForUpdates] → checking
checking → [update-available event] → available
checking → [update-not-available event] → not-available
available → [downloadUpdate] → downloading
downloading → [update-downloaded event] → ready
ready → [installAndRestart] → (app restarts)
* → [error event] → error
```

**Acceptance Criteria:**
- [ ] Khi status=`idle`: hiển thị version hiện tại + "Check for Updates" button
- [ ] Khi status=`checking`: button disabled + spinner
- [ ] Khi status=`available`: hiển thị version mới + "Download" button
- [ ] Khi status=`downloading`: progress bar 0–100%
- [ ] Khi status=`ready`: "Install & Restart" button (pill, primary)
- [ ] Khi status=`error`: error message + "Retry" button
- [ ] IPC listener được cleanup khi component unmount
- [ ] Không crash khi `window.electronAPI` undefined (web fallback)

# 9. Edge Cases

| # | Scenario | Expected Behavior | Handled In |
|---|----------|-------------------|------------|
| 1 | `checkForUpdates()` gọi trong dev mode | Trả mock `UpdateCheckResult { status: 'not-available' }`, không crash | TASK-002 |
| 2 | Mạng mất kết nối khi check updates | `status: 'error'` với message "Network error", có Retry button | TASK-005 |
| 3 | Mạng mất kết nối khi đang download | Dừng progress, `status: 'error'`, không corrupt partial download | TASK-005 |
| 4 | `window.electronAPI` undefined (chạy trong browser) | Hook trả `status: 'idle'`, không crash | TASK-005 |
| 5 | Version hiện tại = version mới nhất | `status: 'not-available'`, hiển thị "You're up to date" | TASK-005 |
| 6 | User click "Check" nhiều lần nhanh | Debounce — chỉ gọi IPC một lần khi đang `checking` | TASK-005 |
| 7 | Component unmount khi đang download | Cleanup IPC listener, không memory leak | TASK-005 |
| 8 | GitHub release không có file cho platform hiện tại | `status: 'error'` với message rõ ràng | TASK-002 |
| 9 | electron-builder không có `GH_TOKEN` | Build fail sớm với message hướng dẫn set env var | TASK-006 |
| 10 | Display-xl font quá lớn trên mobile | Responsive: 86px → ~48px tại 560px breakpoint | TASK-003/004 |
